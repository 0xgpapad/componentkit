Intro

CKSwift is a Swift evolution on ComponentKit which is facilitating a smooth transition to Swift by providing an idiomatic Swift API directly powered by and compatible with ComponentKit. CKSwift make use of a wide spectrum of Swift features including generics, value types, associated types, protocol extensions, function builders, property wrappers & dynamic member lookup.

The core principles for CKSwift are:

  • Idiomatic: Exposing the existing API like-for-like in Swift isn’t sufficient. CKSwift is to leverage Swift language features to empower engineers with a more flexible, concise & safer approach to describing UI.
  • Reusability: Reusing existing already written components both as specs or traditional subclasses.
  • No-cost to non-Swift use-cases: There should be no performance impact to apps that don't adopt Swift and exclusively use the Objective C++ API.
  • Alignment with SwiftUI: Aligning some of the concepts with SwiftUI will simplify the transition between the two frameworks.
